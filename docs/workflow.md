# 企画から投稿までの全体ワークフロー

SNS・動画コンテンツを、企画から公開・投稿まで漏れなく進めるための手順です。  
ミーティングでは **GitHub のタスク一覧** と **task_board.md** を見ながら、このワークフローに沿って進捗を確認してください。

---

## 全体の流れ（概要）

```
企画 → 台本・企画メモ → 制作・編集 → レビュー → 公開・投稿 → 振り返り
```

| フェーズ | 主なアウトプット | 担当の目安 |
|----------|-------------------|------------|
| 1. 企画 | Issue 作成、構成案 | 運用担当 |
| 2. 台本・企画メモ | 台本ファイル、企画メモ | 運用担当 / 編集者 |
| 3. 制作・編集 | 動画データ、投稿文案 | 編集者 |
| 4. レビュー | フィードバック反映 | 運用担当・編集者 |
| 5. 公開・投稿 | 公開済み URL・日時 | 運用担当 |
| 6. 振り返り | 数値・所感のメモ | 運用担当 |

---

## 1. 企画フェーズ

### やること

1. **Issue を作成する**
   - YouTube 動画 → 「YouTube動画企画」テンプレート（`.github/ISSUE_TEMPLATE/video_planning.md`）
   - X / TikTok 投稿 → 「X/TikTok投稿」テンプレート（`.github/ISSUE_TEMPLATE/sns_post.md`）
2. **企画概要・ターゲット・構成案** を Issue に記入する。
3. **task_board.md** の「未着手」にタスクを 1 行追加し、Issue 番号をリンクする。
4. **Assignee** を設定し、期限があれば Issue のタイトルや本文に明記する。

### 完了の目安

- Issue に企画の目的・構成・必要な素材が書かれている。
- 担当と期限が共有されている。

---

## 2. 台本・企画メモフェーズ

### やること

1. **台本または企画メモを作成する**
   - YouTube 長尺 → `contents/youtube/` に `template_script.md` をコピーし、企画名を付けて保存（例: `2026-02_〇〇企画_script.md`）。
   - ショート動画 → `contents/short_videos/` に `template_idea.md` をコピーし、同様に保存。
2. **構成・セリフ・画面指示** を埋め、必要に応じて編集者と共有する。
3. Issue に「台本ドラフト完了」などとコメントし、**task_board.md** の該当タスクを「進行中」に移動する。

### 完了の目安

- 台本 or 企画メモがレビュー可能な状態になっている。
- 編集者が「何を作るか」を把握できる。

---

## 3. 制作・編集フェーズ

### やること

1. **撮影・収録** または **既存素材の編集** を進める。
2. 素材の受け渡しは、共有フォルダ・Issue のコメント・リンクなど、チームで決めた方法で行う。
3. 編集が進んだら Issue に進捗をコメントし、**task_board.md** では「進行中」のままにする。
4. **下書き・初稿** ができたら、次の「レビュー」に進む。

### 完了の目安

- 動画の下書き or 投稿文案・メディアのたたきが用意されている。

---

## 4. レビューフェーズ

### やること

1. **task_board.md** の該当タスクを「レビュー中」に移動する。
2. レビュー担当者に確認を依頼する（Issue でメンション可）。
3. フィードバックを Issue のコメントにまとめ、編集者が修正する。
4. 問題なければ「公開・投稿」フェーズへ進む。

### 完了の目安

- タイトル・サムネ・説明文・投稿テキストなどが確定している。
- 公開・投稿してよいと合意されている。

---

## 5. 公開・投稿フェーズ

### やること

1. **公開・投稿を実行する。**
2. **公開済み URL・投稿日時** を Issue にコメントで記録する。
3. **task_board.md** の該当タスクを「完了」に移動し、チェックを入れる。
4. Issue を「Close」するか、ラベルで「完了」に振り分ける（運用ルールに合わせる）。

### 完了の目安

- コンテンツが公開・投稿されている。
- タスクボードと Issue が完了状態で更新されている。

---

## 6. 振り返り（任意）

### やること

1. 再生数・いいね数・コメントなど、取得できる数値を Issue のコメントや別メモに記録する。
2. 「次に活かす点」「やらなくてよかったこと」を短くメモする。
3. 必要に応じて SOP やテンプレート（`docs/` や `contents/`）を更新する。

---

## ミーティングでの進め方

1. **開始時**: GitHub の Issue 一覧 と **task_board.md** を開き、「未着手」「進行中」「レビュー中」のタスクを確認する。
2. **進行中**: 各タスクの担当・期限・ブロッカーを話し合い、必要なら Issue や task_board をその場で更新する。
3. **終了前**: 次回までにやること（Action Items）を Issue または task_board に明記する。
4. **議事録**: 会話内容を記録し、編集者に読ませたうえで構成を整え、GitHub に同期する（議事録の運用方針に従う）。

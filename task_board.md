ミーティングのたびにこのファイルと GitHub の Issue / Projects を照らし合わせて更新してください。

**凡例**: `[ ]` 未完了 / `[x]` 完了 | **担当** | **期限** | **Issue #**

## 📋 種別の説明

| 種別 | 内容 |
|------|------|
| **YouTube** | 長尺・ショート含む YouTube 動画の企画・台本・編集・公開 |
| **SNS** | X（Twitter）・TikTok 等の投稿（テキスト・メディア・ハッシュタグ） |

---

## 更新ルール

1. 新規タスクは **未着手** に追加し、対応する Issue を作成してリンクする。
2. 着手したら **進行中** に移動する。
3. 編集・下書きができたら **レビュー中** に移動し、確認依頼を行う。
4. 公開・投稿が完了したら **完了** に移動し、チェックを入れる。
5. 完了タスクは定期的にアーカイブし、このシートの行数が増えすぎないようにする（必要なら別ファイルに移す）。
